# 機械部門 学科試験演習アプリ

技能競技大会「機械部門」の学科試験対策用Webアプリです。

スマートフォンでの利用を前提に、○×問題、択一問題、本番模擬、誤答復習、学習履歴確認、問題一覧確認を行えるようにしています。

## 対象

- 技能競技大会
- 機械部門
- 学科試験対策
- スマートフォン利用を想定

## 主な機能

### 演習

- ○×演習
- 択一演習
- 本番模擬
- 誤答復習
- 間違えた問題だけ再演習

### 確認

- 学習履歴確認
- 問題一覧
- 問題詳細表示
- バージョン情報表示

## 出題形式

対応している出題形式は以下です。

- ○×問題
- 択一問題

計算問題は現段階ではアプリ対象外です。  
`questions.json` 内で `isCalculation: true` の問題は、本番模擬では除外されます。

## 本番模擬の仕様

本番模擬は以下の構成を基本とします。

| 種別 | 問題数 |
|---|---:|
| ○×問題 | 60問 |
| 択一問題 | 10問 |
| 合計 | 70問 |

本番模擬では、カテゴリ名は演習中に表示しません。  
また、回答後は自動で次の問題へ進みます。

本番模擬開始時には確認ダイアログを表示します。  
本番模擬中にトップへ戻る場合も、中断確認ダイアログを表示します。

## 採点仕様

採点は以下の方式です。

| 種別 | 正解 | 不正解 | 無回答 |
|---|---:|---:|---:|
| ○×問題 | +0.2 | -0.2 | 0 |
| 択一問題 | +0.4 | -0.4 | 0 |

合計点がマイナスになった場合も、0点には丸めません。  
アプリ上ではマイナス点をそのまま表示します。

## 学習履歴の仕様

学習履歴はブラウザの `localStorage` に保存します。

保存される主な情報は以下です。

- 問題ごとの回答回数
- 正解数
- 不正解数
- 無回答数
- 最終回答結果
- 最終回答日時
- 連続正解数
- 誤答復習対象
- 本番模擬の実施履歴

### 誤答復習対象

不正解または無回答だった問題は、誤答復習対象になります。

誤答復習対象の問題は、2回連続で正解すると対象から外れます。

### 履歴リセット

学習履歴画面では、以下の範囲でリセットできます。

- 全履歴リセット
- 本番模擬履歴のみリセット
- 誤答復習対象のみクリア

## 誤答復習の出題順

誤答復習では、出題順を選択できます。

- 正答率が低い順
- 最後に間違えた順
- 不正解・無回答が多い順
- ランダム
- 登録順

## 通常演習の出題順

○×演習、択一演習では、演習設定画面で出題順を選択できます。

- ランダム
- 登録順
- 正答率が低い順
- 回答回数が少ない順

## 問題一覧

問題一覧では以下の確認ができます。

- キーワード検索
- カテゴリ複数選択
- 形式による絞り込み
- 問題画像ありの絞り込み
- 解説画像ありの絞り込み
- カテゴリ未設定の絞り込み
- 問題詳細表示

問題詳細では、以下を確認できます。

- 問題ID
- 形式
- カテゴリ
- サブカテゴリ
- タグ
- 問題文
- 問題画像
- 正解
- 選択肢
- 解説
- 解説画像

スマートフォンでも読みやすいように、詳細情報はカード形式で表示します。

## 問題データ

問題データは以下に配置します。

```text
public/data/questions.json
```

## questions.json の基本構造

```json
{
  "questionCount": 1,
  "questions": [
    {
      "id": "TF-001",
      "type": "true_false",
      "category": "機械一般",
      "subCategory": "",
      "question": "ここに問題文を入力します。",
      "answer": true,
      "choices": [],
      "explanation": "ここに解説を入力します。",
      "image": "",
      "explanationImage": "",
      "shuffleChoices": false,
      "isCalculation": false,
      "tags": []
    }
  ]
}
```

## ○×問題の例

```json
{
  "id": "TF-001",
  "type": "true_false",
  "category": "機械一般",
  "subCategory": "",
  "question": "安全作業では、保護具の着用が重要である。",
  "answer": true,
  "choices": [],
  "explanation": "安全作業では、作業内容に応じた保護具の着用が必要です。",
  "image": "",
  "explanationImage": "",
  "shuffleChoices": false,
  "isCalculation": false,
  "tags": ["安全"]
}
```

## 択一問題の例

```json
{
  "id": "MC-001",
  "type": "multiple_choice",
  "category": "品質関係",
  "subCategory": "",
  "question": "品質管理で用いられるQC七つ道具として正しいものはどれか。",
  "answer": null,
  "choices": [
    {
      "id": "A",
      "text": "ヒストグラム",
      "isCorrect": true,
      "explanation": "ヒストグラムはQC七つ道具の一つです。"
    },
    {
      "id": "B",
      "text": "貸借対照表",
      "isCorrect": false,
      "explanation": "貸借対照表は会計書類です。"
    }
  ],
  "explanation": "QC七つ道具には、パレート図、特性要因図、ヒストグラムなどがあります。",
  "image": "",
  "explanationImage": "",
  "shuffleChoices": true,
  "isCalculation": false,
  "tags": ["品質管理", "QC"]
}
```

## 画像ファイル

問題画像、解説画像を使用する場合は、`public` 配下に画像を配置し、`questions.json` にパスを記載します。

例：

```text
public/images/sample.png
```

`questions.json` での指定例：

```json
{
  "image": "/images/sample.png",
  "explanationImage": "/images/sample_explanation.png"
}
```

画像を使用しない場合は、空文字にします。

```json
{
  "image": "",
  "explanationImage": ""
}
```

## ローカル起動手順

### 1. リポジトリを取得

```bash
git clone <repository-url>
cd kikai-gakka-exam
```

### 2. パッケージをインストール

```bash
npm install
```

### 3. 問題データを配置

以下の場所に `questions.json` を配置します。

```text
public/data/questions.json
```

### 4. 開発サーバーを起動

```bash
npm run dev
```

表示されたURLをブラウザで開きます。

通常は以下のようなURLです。

```text
http://localhost:5173/
```

## ビルド

公開用ファイルを作成する場合は、以下を実行します。

```bash
npm run build
```

ビルド結果は以下に出力されます。

```text
dist/
```

## プレビュー

ビルド結果をローカルで確認する場合は、以下を実行します。

```bash
npm run preview
```

## GitHub Pages で公開する場合

ViteプロジェクトをGitHub Pagesで公開する場合は、リポジトリ名に応じて `vite.config.js` の `base` 設定が必要になる場合があります。

例：

```js
export default defineConfig({
  base: "/kikai-gakka-exam/",
  plugins: [react()],
});
```

公開前に以下を確認してください。

- `public/data/questions.json` が存在すること
- 画像パスが正しいこと
- `npm run build` が成功すること
- GitHub Pages の公開元が正しく設定されていること

## localStorage について

このアプリの学習履歴は、利用中のブラウザに保存されます。

そのため、以下の場合は履歴が共有されません。

- 別の端末で利用する場合
- 別のブラウザで利用する場合
- ブラウザの保存データを削除した場合

現時点では、履歴データのエクスポート・インポート機能は実装対象外です。
